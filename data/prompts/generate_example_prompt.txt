You are a senior database engineer creating Text2SQL training examples for a banking agent system.

## Context

The database is a Vietnamese banking system (TrustFlow Banking Agent) with 16 tables and relationships:

### Tables:
1. customers (cif_no PK business key) — master data khách hàng
2. accounts (account_no unique, cif_no FK→customers) — tài khoản ngân hàng
3. cards (card_id PK, cif_no FK→customers, account_no FK→accounts) — thẻ debit/credit
4. beneficiaries (beneficiary_id PK, cif_no FK→customers) — người nhận chuyển tiền đã lưu
5. merchants (merchant_id PK) — đơn vị chấp nhận thẻ
6. billers (biller_id PK) — nhà cung cấp dịch vụ (điện, nước, internet)
7. customer_biller_accounts (cif_no FK→customers, biller_id FK→billers) — liên kết KH-biller
8. transaction_categories (category_id PK) — danh mục giao dịch
9. transactions (cif_no FK→customers, account_no FK→accounts, card_id FK→cards, merchant_id FK→merchants, biller_id FK→billers, beneficiary_id FK→beneficiaries, category_id FK→transaction_categories) — lịch sử giao dịch
10. action_requests (cif_no FK→customers) — yêu cầu hành động từ agent
11. api_call_logs (action_id FK→action_requests) — log gọi API
12. audit_logs (action_id FK→action_requests, cif_no FK→customers) — log kiểm toán
13. fraud_reports (reporter_cif_no FK→customers, transaction_ref FK→transactions) — báo cáo lừa đảo
14. reported_accounts (account_no + bank_code unique) — TK bị báo cáo (aggregate risk)
15. reported_customers (cif_no FK→customers) — CIF bị báo cáo (aggregate risk)
16. fraud_decisions (action_id FK→action_requests) — quyết định screening khi chuyển tiền

## Rules

1. **Always use explicit column names** — never SELECT *
2. **Always include LIMIT** (default 100 for lists, can omit for single-row aggregations or when result set is inherently small like GROUP BY on enum)
3. **SELECT only** — no INSERT, UPDATE, DELETE, DDL
4. **Explain JOIN logic clearly** — each JOIN must have a reason tied to the user's question
5. **Use Vietnamese for questions** — the system serves Vietnamese banking customers
6. **Amount is in VND** (đồng), stored as BIGINT
7. **Time filters** use `transaction_time` for transactions, `created_at` for other tables
8. **cif_no is the universal customer key** — used across all customer-related tables
9. **Nullable FK columns** — transactions.card_id is NULL unless CARD_PAYMENT, merchant_id is NULL unless CARD_PAYMENT, biller_id is NULL unless BILL_PAYMENT, beneficiary_id is NULL unless BANK_TRANSFER. JOINs on these should use LEFT JOIN or filter by transaction_type first.
10. **Avoid double-counting** — when JOINing one-to-many, be careful with aggregations
11. **Date functions** — use PostgreSQL syntax: DATE_TRUNC, INTERVAL, CURRENT_DATE, EXTRACT
12. **No PII in examples** — use placeholder cif_no like 'CIF000001'

## Complexity Guidelines

- **simple**: 1 table, basic WHERE/ORDER BY/LIMIT
- **medium**: 2 tables JOIN, aggregation (COUNT/SUM/AVG), GROUP BY
- **complex**: 3+ tables, subqueries, window functions, HAVING, complex date logic, CASE WHEN, CTE
